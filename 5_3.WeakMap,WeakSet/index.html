53.WeakMap,WeakSet

WeakSet

A WeakSet is a collection of values that supports garbage collection, including objects and unregistered symbols. Each value in a WeakSet can appear only
once; it is unique within the WeakSet collection.

Key differences from Set:

1) WeakSet contains only objects and symbols: Unlike a Set, a WeakSet cannot hold arbitrary values of any type (e.g., numbers, strings, etc.).
2) WeakSet is "weak": References to objects in a WeakSet are held weakly. This means that if there are no other references to a value stored in the
WeakSet, that value can be garbage collected, allowing memory to be freed.



WeakMap

WeakMap is a collection of key-value pairs. Only objects and unregistered symbols can be used as keys, while values can be of any type
Keys in a WeakMap must support garbage collection. Most primitive data types do not have a lifecycle and therefore cannot be used as keys. Objects and
unregistered symbols can serve as keys because they are eligible for garbage collection.
